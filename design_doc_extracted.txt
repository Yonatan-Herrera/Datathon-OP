1. Overview
This document outlines the design and implementation plan for the Product Track submission of the DataGood Data Analytics Case Competition. The deliverable is an interactive dashboard enabling policymakers and foundation leaders to explore global philanthropy funding patterns using the OECD dataset — over 110,000 international funding transactions spanning multiple years, countries, sectors, and donors.
1.1 Tech Stack
Tableau Public — primary dashboard and publishing platform
Python / pandas — data preprocessing and cleaning pipeline
GitHub — version control and submission artifact hosting
2. Problem Statement
Global philanthropic funding is complex and fragmented. Policymakers and foundation leaders need a clear, navigable view of where funding flows — across geographies, sectors, time periods, donors, and recipients — to make strategic decisions and identify gaps. The OECD dataset provides a rich, standardized view of this landscape, but its raw form is inaccessible to non-technical users.
The dashboard must answer questions like:
Which foundations are the top donors out of a given country?
Which 5 countries receive the most funding for maternal health?
How has global funding for climate change evolved over time?
Which donor contributes most to infectious disease funding in India?
2.1 Tech Stack

Python / pandas — initial exploration to confirm the dataset can answer each target question
Tableau Public — the interface through which end users pose and answer these questions themselves

5. Dashboard Architecture
The dashboard is organised into four interconnected views, each serving a distinct analytical purpose. All views share a global filter panel so users can quickly slice by year range, region, sector, and donor country.
5.1 Global Filters (Persistent Across All Views)
Year / Time Range — slider or dropdown (single year or multi-year range; note: some foundations report 2020-2023 aggregate)
Donor Country — filter by the foundation's country of legal base
Region / Country — filter by recipient geography at macro or country level
Sector — filter by OECD CRS sector (e.g., Health, Education, Environment)
Type of Flow — cross-border vs. domestic philanthropy
5.2 View 1: Global Overview Map
Purpose: Immediate geographic orientation — show where money goes and how much.
Choropleth world map coloured by total disbursements to each recipient country
Hover tooltip shows: country name, total USD received, top sector, top donor
Toggle between absolute amounts and per-capita funding (where population data available)
Bar chart below map: Top 10 recipient countries ranked by disbursements
5.3 View 2: Sector & Thematic Deep-Dive
Purpose: Explore how funding is distributed across causes and flag thematic priorities.
Treemap or stacked bar: funding by sector, drillable to subsector
Line chart: funding trend over time for selected sector(s)
Thematic overlays: toggle to highlight grants scoring 1 or 2 on gender, climate mitigation, climate adaptation, biodiversity, or nutrition markers
SDG filter: show only grants tied to specific Sustainable Development Goals
5.4 View 3: Donor Leaderboard
Purpose: Identify which foundations are the largest contributors and where they give.
Ranked table of top N donors by total disbursements (configurable N)
Clicking a donor filters the map and sector views to show only that foundation's grants
Donor profile card: country of origin, total giving, primary sectors, top recipient countries
Comparison mode: select 2-3 donors side by side
5.5 View 4: Recipient & Project Explorer
Purpose: Discover which organisations and projects receive the most funding and why.
Searchable, sortable table of grant recipients with columns: recipient name, total USD, sector, donor, country, year
Filter by channel of delivery (NGO, multilateral, government, etc.)
Expandable rows show project description and expected duration

7. Data Preparation Notes
The OECD dataset requires some preprocessing before loading into a dashboard tool:
Deflated USD values (usd_disbursements_defl) should be used as the primary metric — these are already normalised to 2023 constant dollars for fair year-over-year comparison.
Projects with multiple sector codes are split at the sector level (tracked by row_id). Aggregate these rows when computing total project funding to avoid double-counting.
Foundations with NDA restrictions report data only in aggregate form with year = '2020-2023'. Treat these as a single time bucket and exclude from year-over-year trend calculations.
Null values in marker columns (gender_marker, climate_change_mitigation, etc.) indicate non-screened projects — exclude nulls from thematic filter counts to avoid misleading zero results.
The channel_reported_name field may differ from channel_name (standardised). Use channel_name for consistent grouping; show channel_reported_name in project detail views.
7.1 Tech Stack
Python / pandas — primary cleaning tool: deduplication on row_id, sector-split aggregation, null flagging, and USD deflation validation
openpyxl — read raw OECD XLSX preserving multi-sheet structure before flattening to a single clean CSV
Jupyter Notebook — reproducible cleaning pipeline with inline checks; exported as HTML for documentation
Git — tracks each cleaning step so any data decision can be reviewed or rolled back
